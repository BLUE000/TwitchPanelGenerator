# 要件定義書 (Requirements Definition)

## 1. システムの目的
Twitchのプロフィール欄に使用するパネル画像（幅320px）を、画像編集ソフトを持たないユーザーでもWebブラウザ上で直感的に作成できるツールを提供する。

## 2. ターゲットユーザー
- 画像編集ツール（Photoshop等）の操作に不慣れな配信者。
- ブラウザのみで手軽に見栄えの良いパネルを作成したいユーザー。

## 3. 機能要件
### 3.1. 背景画像合成
- ユーザーが手元の画像を背景としてアップロード（ブラウザメモリ保持）できる。
  - 画像選択プルダウン（コンテンツ画像）と追加用の「追加」ボタンは横並びのレイアウトで配置する。
- 幅320pxに自動リサイズされ、縦幅はテキスト量に応じて自動延長（タイリング）される。
- 手動での高さ指定も可能。

### 3.2. ヘッダ画像機能
- 背景とは別に、上部に配置するヘッダ画像を指定できる（ON/OFF可能）。
  - 画像選択プルダウン（ヘッダ画像）と追加用の「追加」ボタンは横並びのレイアウトで配置する。
- 幅320pxに自動リサイズされて最上部に配置される。
- ヘッダ画像上に、専用のテキスト（配置、フォント、色、縁取り指定可能。Markdown簡易装飾のインライン装飾も対応）を合成できる。

### 3.3. テキスト合成（本文）
- 複数行のテキストを入力し、背景画像上に合成できる。
  - 入力用のテキストフィールド（ヘッダテキストおよび本文テキスト）は、テキスト設定の内部にある「🔤 表示文字」というサブアコーディオン（アコーディオン内のネスト）内に集約して配置する。
  - ヘッダテキスト入力欄は、ヘッダ画像が有効な場合のみ「表示文字」サブアコーディオン内に表示される。
  - 「ヘッダ装飾設定」および「本文装飾設定」も、テキスト設定の内部に個別のサブアコーディオン（アコーディオン内のネスト）としてそれぞれ配置し、UIを階層化・整理する。
- テキストは幅320pxのパディング内ではみ出さないよう自動で折り返される。
- フォントサイズ、行間、段落間、配置（左・中央・右）、文字色、縁取りのカスタマイズが可能。
- **ドラッグ位置調整機能**: プレビュー画像内の本文テキスト部分をドラッグ（タッチ操作含む）することで、本文の合成位置（X座標・Y座標）を視覚的に調整できる。ドラッグに応じて設定フォームの「X座標」「Y座標」インプットが同期して更新される。
- **Markdown簡易装飾機能**: 行頭の記号およびインライン修飾子に応じて、以下の装飾を自動で適用する。
  - **行頭のブロック装飾**:
    - `# ` (見出し1): 文字サイズが1.5倍になり、下部に余白が追加される。
    - `## ` (見出し2): 文字サイズが1.25倍になる。
    - `- ` または `* ` (リスト): 先頭が「・」に変換され、綺麗に字下げ（インデント）される。
    - `> ` (引用): 左側にアクセントの縦線が描画され、テキスト全体が字下げされる。
    - `---` (横線): 幅いっぱいに区切り線が描画される。
  - **インライン装飾**:
    - `**テキスト**` または `__テキスト__` (太字): 該当箇所が太字（Bold）で描画される。
    - `*テキスト*` または `_テキスト_` (斜体): 該当箇所が斜体（Italic）で描画される。
    - `***テキスト***` (太字斜体): 該当箇所が太字かつ斜体（Bold Italic）で描画される。
    - `<u>テキスト</u>` または `++テキスト++` (下線): 該当箇所に下線（Underline）が引かれる（ヘッダ・本文両対応）。
    - `{色名または番号またはカラーコード:テキスト}` (部分文字色変更): 該当箇所の文字色が指定された色で描画される（ヘッダ・本文両対応）。
      - プリセット色（赤、青、黄、緑、紫、オレンジ、ピンク、水色、白、黒）の日本語・英語表記に対応。
      - 設定パネルで設定した3つの「カスタム部分色」（1〜3）に対応。
      - 直接指定されたカラーコード（例：`#ff00aa`）に対応。
      - タグは入れ子（ネスト）に対応。

### 3.4. フォント管理
- 標準フォント（Arial, Meiryo等）をデフォルトで提供。サーバーに存在しない場合はGoogle Fontsから代替フォントを自動ダウンロードしてキャッシュする。
- ユーザーが独自のフォントファイル（.ttf, .otf, .ttc）をアップロード（ブラウザメモリ保持）して適用できる。

### 3.5. プレビューと保存
- 設定内容を即座にプレビュー確認できる。
- **テキストレイヤ枠表示機能**: プレビューの下に「枠表示: ON/OFF」を切り替えるトグルボタンを設置し、ONの場合は本文テキストの描画範囲を点線の矩形（枠線）でプレビュー上に重ねて表示する。これにより、ドラッグ位置調整時や配置確認時にレイヤー位置を視覚的に分かりやすくする。
- 完成した画像をPNG形式でローカルデバイスへダウンロードできる。枠線はプレビュー上でのみ描画され、**PNG保存時には出力されない**。

## 4. 非機能要件
### 4.1. プライバシーとセキュリティ（完全非保存化）
- **ファイル非保存**: ユーザーが追加した画像やフォントファイルは、サーバーに一切送信せず、ブラウザのローカルメモリ上でのみ処理する。
- **メモリ処理**: 画像合成はブラウザ上の HTML5 Canvas と JavaScript メモリ上でのみ行われ、サーバーとの通信は発生しない。
- データのライフサイクルは「ブラウザを閉じたら終了」とする。

### 4.2. 稼働環境
- PHP 7.4〜8.x が動作するWebサーバー（さくらのレンタルサーバ等。HTML配信および設定CSV保存処理のみ使用）。
- HTML5 Canvas およびモダンな JavaScript API (FontFace API, URL.createObjectURL 等) が動作するWebブラウザ。
- 1ファイル（`twitch_panel_generator.php`）のみで完結するデプロイ容易性。
