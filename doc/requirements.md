# 要件定義書 (Requirements Definition)

## 1. システムの目的
Twitchのプロフィール欄に使用するパネル画像（幅320px）を、画像編集ソフトを持たないユーザーでもWebブラウザ上で直感的に作成できるツールを提供する。

## 2. ターゲットユーザー
- 画像編集ツール（Photoshop等）の操作に不慣れな配信者。
- ブラウザのみで手軽に見栄えの良いパネルを作成したいユーザー。

## 3. 機能要件
### 3.1. 背景画像合成
- ユーザーが手元の画像を背景としてアップロード（ブラウザメモリ保持）できる。
- 幅320pxに自動リサイズされ、縦幅はテキスト量に応じて自動延長（タイリング）される。
- 手動での高さ指定も可能。

### 3.2. ヘッダ画像機能
- 背景とは別に、上部に配置するヘッダ画像を指定できる（ON/OFF可能）。
- 幅320pxに自動リサイズされて最上部に配置される。
- ヘッダ画像上に、専用のテキスト（配置、フォント、色、縁取り指定可能）を合成できる。

### 3.3. テキスト合成（本文）
- 複数行のテキストを入力し、背景画像上に合成できる。
- テキストは幅320pxのパディング内ではみ出さないよう自動で折り返される。
- フォントサイズ、行間、段落間、配置（左・中央・右）、文字色、縁取りのカスタマイズが可能。

### 3.4. フォント管理
- 標準フォント（Arial, Meiryo等）をデフォルトで提供。サーバーに存在しない場合はGoogle Fontsから代替フォントを自動ダウンロードしてキャッシュする。
- ユーザーが独自のフォントファイル（.ttf, .otf, .ttc）をアップロード（ブラウザメモリ保持）して適用できる。

### 3.5. プレビューと保存
- 設定内容を即座にプレビュー確認できる。
- 完成した画像をPNG形式でローカルデバイスへダウンロードできる。

## 4. 非機能要件
### 4.1. プライバシーとセキュリティ（完全非保存化）
- **ファイル非保存**: ユーザーが追加した画像やフォントファイルは、サーバーのディスク上に一切保存しない（一時ファイルすら作成しない）。
- **メモリ処理**: 画像合成はサーバーのメモリ上（出力バッファ）でのみ行い、処理完了後即座に破棄する。
- データのライフサイクルは「ブラウザを閉じたら終了」とする。

### 4.2. 稼働環境
- PHP 7.4〜8.x が動作する標準的なWebサーバー（さくらのレンタルサーバ等）。
- GDライブラリ（FreeTypeサポート必須）が有効であること。
- 1ファイル（`twitch_panel_generator.php`）のみで完結するデプロイ容易性。
