# 詳細設計書 (Detailed Design)

## 1. フロントエンド実装詳細

### 1.1. メモリ内ファイル管理機構
サーバーへの永続的なファイルアップロードを避けるため、ブラウザ側でファイルを一時管理する。
```javascript
const localFiles = {
    bgImages: {},     // { "file_1": File Object, ... }
    headerImages: {}, // { "file_2": File Object, ... }
    fonts: {}         // { "file_3": File Object, ... }
};
let fileCounter = 1;
```
- `<input type="file">` の `change` イベントで取得したファイルを上記オブジェクトに格納する。
- 同時に、対応する `<select>` 要素に `<option>` を動的に追加し、選択状態にする。

### 1.2. 動的フォント登録機構
- ユーザーがカスタムフォント（.ttf / .otf / .ttc）を選択した際、以下の JavaScript 処理を用いてブラウザに動的にフォントを適用する。
```javascript
const reader = new FileReader();
reader.onload = function(e) {
    const fontFace = new FontFace(fontFamilyName, e.target.result);
    fontFace.load().then(function(loadedFace) {
        document.fonts.add(loadedFace);
    });
};
reader.readAsArrayBuffer(fileObj);
```

### 1.3. UIレイアウト構成
- **画像選択と追加ボタンの並列配置**:
  - 各画像設定（ヘッダ画像、コンテンツ画像）について、選択中の画像を切り替える `<select>` 要素と、ローカル画像を追加するボタン（実際は `<input type="file" style="display: none;">` をクリックするボタン）を、フレックスボックス（`display: flex; gap: 10px;`）を使用して1行に並べて配置する。これにより、UIの縦方向の専有面積を抑え、画像追加と切り替えの操作性を向上させる。
- **テキスト設定の階層構造（サブアコーディオン）**:
  - テキスト設定アコーディオンの直下に、文字入力を司る「🔤 表示文字」サブアコーディオン（インナー `<details>`）を配置する。
  - その中に、ヘッダテキスト用の `<input type="text" id="headerTextContent">` と、本文テキスト用の `<textarea id="textContent">` を格納し、文章入力を集約する。
  - また、装飾設定部分も「🎨 ヘッダ装飾設定」および「⚙️ 本文装飾設定」としてサブアコーディオン（インナー `<details>`）化し、テキスト設定内部を3つのモジュール式サブアコーディオン構造に階層化・整理する。
  - ヘッダテキストの入力欄を囲むコンテナ（`#headerTextInputArea`）およびヘッダ装飾設定（`#headerTextSettingsArea`）は、ヘッダ画像の有効・無効切り替えトグル（`#headerEnabled`）のイベントハンドラに連動して表示・非表示を切り替える。

### 1.4. 本文テキストのドラッグ位置調整機構
- **バウンディングボックスの算出**:
  - 本文テキスト全体の縦方向の描画範囲を、`drawCanvas` でのレイアウト計算時に更新する。
  - 本文の開始Y座標 `minY` は `textY + headerHeight` となり、終了Y座標 `maxY` は `requiredHeight + headerHeight` となる。
  - 本文テキストが空でなく、有効にレンダリングされている場合に限り、バウンディングボックスのアクティブフラグ（`bodyTextDragArea.active`）を `true` に設定する。
- **相対座標の補正とドラッグ制御**:
  - プレビュー画像（`#previewContainer img`）上での `mousedown` / `touchstart` イベントをトリガーとし、ポインタの位置が `bodyTextDragArea` のY座標の範囲内にあるか検証する。
  - 画像要素が表示サイズに応じて拡大縮小されている可能性を考慮し、バウンディングボックス判定には、イベントのクライアント座標を実 Canvas の解像度比率（`TARGET_WIDTH` 対 画像の表示横幅）に基づいてスケール補正した Canvas 上の座標に変換して使用する。
  - ドラッグ（`mousemove` / `touchmove`）中は、ポインタの移動差分量を Canvas 上の座標変化量にスケール補正して新たな `textX` / `textY` を算出し、フォームのインプット値を書き換える。
  - 操作感の向上と負荷軽減のため、ドラッグ中は `updatePreview` を経由した「生成中...」プレースホルダーの再レンダリング処理をバイパスし、`drawCanvas` で同期生成した Canvas の Base64 データを直接 `<img>` 要素の `src` へ即時注入することで、チラつきのない滑らかなリアルタイム追従を実現する。
  - ドラッグ終了（`mouseup` / `touchend`）時に設定値を LocalStorage に保存し、最終的なプレビュー同期を完了する。

### 1.5. テキストレイヤ枠表示機構
- **トグル状態管理**:
  - グローバル変数 `showTextFrame`（初期値 `false`）を定義し、「枠表示: ON/OFF」ボタン（`#toggleFrameBtn`）をクリックした際、`toggleTextFrame` 関数を呼び出して状態を反転させる。
  - トグル状態に応じて、ボタンのラベルテキスト（`枠表示: ON` または `枠表示: OFF`）およびボタンスタイル（`btn-primary` または `btn-secondary`）を動的に切り替える。
- **枠線のリアルタイム描画とPNG保存時の制御**:
  - `drawCanvas(showFrame)` は第一引数 `showFrame`（デフォルト `false`）を受け取るよう拡張する。
  - レンダリング時、`showFrame` が `true` で、かつ `bodyTextDragArea.active` が `true` の場合、本文テキスト全体の描画範囲（X方向：全行の描画開始X座標の最小値 `minX` から描画終了X座標の最大値 `maxX`、Y方向：`minY` から `maxY`）を囲む、パディング付き（上下左右2px）の点線枠（`ctx.setLineDash([4, 4])`、枠線色 `#9146ff`、太さ 1.5px）を `ctx.strokeRect` を用いて描画する。
  - `saveImage()` 関数内では `drawCanvas(false)` を明示的（または引数なしのデフォルト値）で呼び出すことで、**PNG保存用に出力される Canvas には点線枠を描画しない**ように制御する。

## 2. フロントエンド画像描画・合成詳細 (`drawCanvas`)

### 2.1. インラインMarkdown解析ロジック (`parseInlineStyles`)
- テキストの各行について、インラインの太字、斜体、下線、および部分文字色を解析するため、文字列を文字トークンの配列に変換する。
- 解析は左から右へ文字を走査して行い、カラー指定の状態を管理するスタック（`colorStack`）と各種フラグを用いて、以下のトークナイズ規則を適用する。
  - `<u>` または `</u>` に遭遇した場合（大文字小文字を区別しない）: 下線フラグ（`underline`）のオン/オフを切り替え、タグの長さに応じて3文字または4文字進む。
  - `++` に遭遇した場合: 下線フラグ（`underline`）のオン/オフをトグルし、2文字進む。
  - `**` または `__` に遭遇した場合: 太字フラグ（`bold`）のオン/オフをトグルし、2文字進む。
  - `*` または `_` に遭遇した場合: 斜体フラグ（`italic`）のオン/オフをトグルし、1文字進む。
  - `{` に遭遇した場合: 対応する `}` の位置を探索し、その中の `:` の左側にある文字列（色名、カスタム色番号 1〜3、または `#RRGGBB` 形式 of カラーコード）から色を解決して `colorStack` にプッシュし、コロンの後ろまで走査を進める。
  - `}` に遭遇し、かつ `colorStack` が空でない場合: `colorStack` から最上位の色をポップし、1文字進む。
  - その他の文字: 現在の `bold` / `italic` / `underline` の状態、および `colorStack` の最上位（存在する場合）のカラーを付与して、文字トークンとして配列に追加し、1文字進む。

### 2.2. Canvas描画プロセス
#### 1. 非同期リソースロード
- `URL.createObjectURL(fileObj)` で背景画像とヘッダ画像の Blob URL を生成し、`HTMLImageElement` を非同期でロードする。
- すべての画像要素の `onload` および使用するカスタムフォントが `document.fonts.ready` になるのを待ってから描画処理を開始する。

#### 2. Canvasサイズ決定とタイリング
- 横幅は `TARGET_WIDTH` (320px) に固定。
- 背景画像は、元画像の縦横比を維持して幅320pxにリサイズしたときの高さを算出する。
- テキスト描画に必要な高さを事前算出（行ごとの文字幅を Canvas コンテキストで測定し折り返し段落数を掛ける）。
- `heightMode === 'manual'` の場合は指定された手動高さを、`'auto'` の場合は背景画像高さとテキスト描画に必要な高さの大きい方を採用する。
- 最終画像高さ = ボディ高さ ＋ ヘッダ高さ（有効時）。
- 背景画像は、ヘッダ画像の直下から画像の終端まで、縦方向にループ描画（タイリング）して配置する。

#### 3. テキストのワードラップ（自動折り返し）と描画
- **ブロック解析**: 行頭の記号（`#`, `##`, `-`, `>`, `---`）に基づいて、段落全体のスタイルを決定する。
  - `h1`: フォントサイズ 1.5倍 ＋ 下部余白
  - `h2`: フォントサイズ 1.25倍
  - `list`: 行頭に「・」を追加し、フォントサイズ分のインデント幅を確保。2行目以降もインデントを適用する。
  - `quote`: インデント幅（フォントサイズ×1.5）を確保し、描画時に左端に縦線を描画する。
  - `hr`: 区切り線描画用のダミー段落。
- **幅測定と折り返し**:
  - 各文字トークンを描画するフォントスタイル（`italic bold 20px ...` 等）を Canvas の `ctx.font` に都度セットし、`ctx.measureText(token.char).width` で文字幅を順次合算する。
  - `ctx.font` の指定時に、不要なスペース（例: 太字や斜体が無効な場合の空値連結による多重スペース）が含まれているとフォント適用が失敗するため、代入直前に `.trim().replace(/\s+/g, ' ')` の正規化を行う。
  - 割り当てられた描画可能幅を超えた時点で、次の文字トークンから新しい行として処理（折り返し）する。
- **描画**:
  - テキスト配置（left/center/right）に基づいて描画開始X座標を計算する。
  - 文字ごとに `ctx.font` を正規化して変更し、さらに文字トークン個別に色が設定されている（部分文字色変更機能）場合は `ctx.fillStyle` をその色に上書きする（設定されていない場合は行全体のデフォルト色）。
  - `ctx.strokeText` (有効時) および `ctx.fillText` を呼び出して描画する。
  - 文字トークンの `underline` フラグが真である場合、文字色（`ctx.fillStyle`）と同色で、ベースライン下部（`line.y + headerHeight + Math.round(fontSize * 0.1)`）に、太さ `Math.max(1, Math.round(fontSize / 15))` の直線を `ctx.moveTo` / `ctx.lineTo` / `ctx.stroke` により描画する。
  - 描画後、その文字の幅だけX座標を右に進める。


#### 4. ヘッダテキストの描画
- ヘッダテキストに対しても、本文と同様に `parseInlineStyles` を適用してインラインMarkdown（太字・斜体・下線・部分文字色）をパースする。
- 全体の描画幅を文字ごとに計測して合算し、指定されたヘッダテキスト配置（中央・左・右）に基づいて描画開始X座標を決定する。
- 文字ごとに `ctx.font` の不要スペース正規化を行い、個別に色が設定されている場合は `ctx.fillStyle` に上書きして、`strokeText`（有効時）および `fillText` を用いて文字単位で水平方向に描画する。
- 文字トークンの `underline` フラグが真である場合、ヘッダ文字色と同じ色で、ベースライン下部（`hDrawY + Math.round(headerFontSize * 0.1)`）に、太さ `Math.max(1, Math.round(headerFontSize / 15))` の直線を引く。

### 2.3. 完全ローカル出力
- `canvas.toDataURL('image/png')` を用いて、生成された画像の Base64 エンコード文字列を取得し、プレビュー要素に反映する。
- 保存時は `canvas.toBlob` でバイナリデータを取得し、動的アンカー要素を用いてサーバーを介さずにブラウザローカルで直接ダウンロード保存を行う。
