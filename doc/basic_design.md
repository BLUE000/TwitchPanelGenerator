# 基本設計書 (Basic Design)

## 1. アーキテクチャ
本システムは `twitch_panel_generator.php` および `index.php` の1ファイルで構成され、フロントエンド（HTML/CSS/JS）が画像合成ロジックを含めたすべての処理を行い、バックエンド（PHP）は初期ページの配信と設定のバックアップ保存のみを補助する。

### 1.1. フロントエンド（ブラウザ側）
- **UI構築**: HTMLと生のCSS（Vanilla CSS）を用いて、レスポンシブかつモダンなUIを提供する。
- **画像合成（HTML5 Canvas）**: サーバー通信を行わず、ブラウザ側の `Canvas API` を使用してメモリ上で合成処理を完結する。
- **状態管理**: 
    - 各種設定値（テキスト、色など）は `localStorage` に自動保存・復元される。
    - ユーザーが選択したファイル（画像、フォント）は JavaScript のグローバルオブジェクト（`localFiles`）内のメモリ上に `File` オブジェクトとして保持される。
    - 選択されたカスタムフォントは、描画時に `FontFace` API を用いてブラウザへ動的に読み込まれる。
- **通信方式**: 画像生成に関する通信は発生しない。設定のCSVインポート・エクスポートのみブラウザローカルで完結する。

### 1.2. バックエンド（サーバー側）
- **エンドポイント**: 初期表示時のHTML/CSS/JSの配信を行う。画像処理ライブラリ（GD等）は一切使用しない。

## 2. 画面構成
画面は左右2ペインのグリッドレイアウト（モバイル時は縦積み）で構成される。
- **左ペイン（設定パネル）**: 
    - 「画像設定」「テキスト設定」「出力設定」のアコーディオン風カードUI。
- **右ペイン（プレビューエリア）**: 
    - 追従型（`position: sticky`）のプレビュー表示領域。
    - 「プレビュー更新」および「PNG保存」のトリガーボタン。

## 3. データフロー
### 3.1. プレビュー実行時
1. ユーザーが「プレビュー更新」を押下。
2. JSが入力値（テキスト、配色、サイズ等）および `localFiles` に保持されている画像ファイル（`File` オブジェクト）を取得。
3. `URL.createObjectURL` を利用して画像を `HTMLImageElement` としてロード。
4. 動的生成した Canvas（幅320px）に対して、背景画像タイリング、ヘッダ画像、各種装飾（枠線、引用線、横線）、テキストの順に描画。
5. テキスト描画時はインラインMarkdownを解析し、太字や斜体に応じて文字ごとにフォントスタイルを切り替えて描画。
6. 描画完了後、`canvas.toDataURL('image/png')` を呼び出し、Base64データをプレビュー用の `<img>` 要素の `src` にセットして画面に描画。

### 3.2. 保存実行時
1. ユーザーが「PNG保存」を押下。
2. プレビュー時と同様に Canvas 上で画像を生成。
3. `canvas.toBlob` を使用して画像データを `Blob` として取得。
4. 動的に生成した `<a>` タグの `href` に `URL.createObjectURL(blob)` を設定し、ローカルデバイスへのダウンロードを発火させる。
