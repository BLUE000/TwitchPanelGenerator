# 基本設計書 (Basic Design)

## 1. アーキテクチャ
本システムは `twitch_panel_generator.php` の1ファイルで構成され、フロントエンド（HTML/CSS/JS）とバックエンド（PHP）の役割を内包する。

### 1.1. フロントエンド（ブラウザ側）
- **UI構築**: HTMLと生のCSS（Vanilla CSS）を用いて、レスポンシブかつモダンなUIを提供する。
- **状態管理**: 
    - 各種設定値（テキスト、色など）は `localStorage` に自動保存・復元される。
    - ユーザーが選択したファイル（画像、フォント）は JavaScript のグローバルオブジェクト（`localFiles`）内のメモリ上に `File` オブジェクトとして保持される。
- **通信方式**: `Fetch API` と `FormData` を使用し、非同期でバックエンドと通信する。ページ遷移は発生しない。

### 1.2. バックエンド（サーバー側）
- **エンドポイント**: 同一ファイルの先頭部分で `$_POST['action']` を判定し、APIとして振る舞う。
- **画像処理**: PHPの標準画像モジュールである `GDライブラリ` を使用。
- **ファイルI/O**: `$_FILES` で受け取った一時ファイル（サーバーが一時ディレクトリに置くファイル）を直接 `imagecreatefrom*` 等で読み込み、ユーザーのディレクトリ内にはファイル保存を一切行わない（完全インメモリ処理）。

## 2. 画面構成
画面は左右2ペインのグリッドレイアウト（モバイル時は縦積み）で構成される。
- **左ペイン（設定パネル）**: 
    - 「画像選択」「Header 画像設定」「フォント選択」「テキスト設定」「縁取り設定」「出力設定」のアコーディオン風カードUI。
- **右ペイン（プレビューエリア）**: 
    - 追従型（`position: sticky`）のプレビュー表示領域。
    - 「プレビュー更新」および「PNG保存」のトリガーボタン。

## 3. データフロー
### 3.1. プレビュー実行時
1. ユーザーが「プレビュー更新」を押下。
2. JSが `FormData` を構築。テキストパラメータに加え、`localFiles` メモリから該当の `File` オブジェクトを付与。
3. Fetch POSTリクエストを発行。
4. PHP側が `generateImage()` を実行。メモリ上で画像を合成。
5. `ob_start()` を用いてPNGバイナリデータを取得し、Base64エンコードしてJSONで返却。
6. JSがBase64データを `<img src="data:image/png;base64,...">` として画面に描画。

### 3.2. 保存実行時
1. ユーザーが「PNG保存」を押下。
2. JSが `FormData` を構築（`action=save`）。
3. Fetch POSTリクエストを発行。
4. PHP側が画像を合成し、`Content-Type: application/octet-stream` と共にバイナリデータを直接ストリーム出力して終了。
5. JSがレスポンスを `Blob` として受け取り、動的に生成した `<a>` タグを用いてローカルダウンロードを発火させる。
